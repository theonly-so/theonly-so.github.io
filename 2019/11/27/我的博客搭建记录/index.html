我的博客搭建记录

记录自己第一次的博客搭建过程，花了大概一个下午的时间，中间也遇到了一些问题，顺便吧解决方法也写下来，以供以后查阅。
以后应该会偶尔在上面记录一些工作、生活和学习方面的收获和感想。

搭建环境

平台：windows10
框架：hexo
网站：github

搭建步骤

  • step1:安装nodejs
    直接搜索nodejs下载最新版本安装即可，这里使用的是12.13.1版本。
  • step2:安装hexo
    在安装hexo之前，需要先安装git工具,安装完成后，打开gitbash的命令行窗口输入以下命令。
    npm i -g hexo
  • step3:新建一个目录用于存放相关的文件
    如E:\blog，cd到该目录下执行hexo init初始化该文件夹
  • step4:本地预览
    hexo clean或hexo c清理
    hexo generate或hexo g生成命令
    hexo sever或hexo s运行，可以在本地查看，默认链接 localhost:4000
  • step5:远程部署
    部署之前需要确保git的用户名和邮箱已经配置完成
    在bash里输入 git config会在目录下生成一个.git文件夹，编辑其中的config文件
    或者直接进行全局配置，输入
    $ git config --global user.name "theonly-so"
    $ git config --global user.email "1459638213@qq.com"
    更改当前目录下的_config.yml文件，在结尾补充以下内容 这里的repo是在github上新建的仓库，仓库的路径必须是 用户名.github.io
    然后执行hexo deploy或hexo d即可发布到github，此时通过theonly-so.github.io即可访问自己的博客了。

更换主题

通过这个链接可以浏览一些自己喜欢的主题，并跳转到对应的github页面，通过git clone ... themes\themename或者直接下载到对应目录下。
然后修改根目录下的_config.yml文件中的theme: icarus来更改主题，更多内容请参考主题的说明文档

撰写博客

hexo new post postname新建一篇博客
hexo new draft draftname新建一篇草稿，草稿别人是看不见的

会分别在source/_posts和source/_drafts文件夹下生成对应名称的文件，通过相应编辑器进行编辑即可

hexo server --draft可以本地查看草稿
hexo publish draftname可以把草稿发布，会移动到source/_posts下
